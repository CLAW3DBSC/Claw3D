## Summary

This PR builds on the runtime/provider seam from PR #89 by adding the
first direct HTTP-backed `custom` runtime path and aligning the office
screen with the provider abstraction instead of assuming every backend is
an OpenClaw-style gateway.

It is intended to move Claw3D forward in two ways:

- make non-gateway orchestrator runtimes usable through a generic
  upstream-safe `custom` provider
- reduce office/chat/bootstrap coupling to raw gateway transport

## Relationship To PR #89

PR #89 remains the Hermes-focused bridge path.

This PR is separate on purpose:

- PR #89 keeps the Hermes adapter stable for upstream review
- this PR continues the runtime seam by adding the first direct HTTP
  provider path and custom-runtime groundwork

## What This PR Includes

### 1. First-class `custom` runtime provider

- adds `custom` as a real provider ID in the runtime seam
- adds provider metadata support
- adds `CustomRuntimeProvider`
- adds provider selection for `custom` alongside OpenClaw, Hermes, and Demo

### 2. Direct HTTP runtime path

- adds a same-origin runtime proxy at `src/app/api/runtime/custom/route.ts`
- routes custom runtime requests through Studio instead of direct browser
  fetches
- probes:
  - `GET /health`
  - `GET /state`
  - `GET /registry`
- sends chat through:
  - `POST /v1/chat/completions`

### 3. Office/runtime alignment

- moves the office bootstrap/chat/model flow onto the provider seam
- reduces direct dependence on raw gateway client calls in the core office
  path
- keeps model loading capability-driven

### 4. Multi-agent focus cleanup

- normalizes local-agent focus behavior in the office
- keeps chat target and selected agent more consistently in sync across:
  - company-builder selection
  - create-agent completion
  - monitor selection
  - GitHub review focus
  - QA testing focus
  - editor-opening flows

### 5. Test coverage updates

- adds `custom` adapter coverage to `useGatewayConnection`
- adds `custom` provider selection coverage to `useRuntimeConnection`

## Current Scope Limits

This PR does **not** claim:

- a Vera-specific built-in upstream provider
- ACP support
- process launch from Studio
- provider-native streaming for `custom`
- full long-lived session persistence semantics for `custom`

The `custom` provider is intentionally conservative in this branch.

## Shortfalls / Follow-Up Work

### Still missing

- streaming event normalization for direct HTTP custom runtimes
- richer session lifecycle handling beyond synthetic provider sessions
- stronger runtime metadata display in the office UI
- full browser smoke and integration test coverage against a live custom
  runtime
- app-level shared runtime connection state

### Regressions guarded against

This branch was built with the Hermes regressions from the shared gateway
path in mind. Reviewer attention should stay on:

- not leaking OpenClaw token assumptions into `custom`
- not routing `custom` through the WebSocket gateway path
- not over-claiming capabilities that the provider does not actually expose
